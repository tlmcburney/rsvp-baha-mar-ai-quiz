# RSVP Quiz App — Session Handoff

## What We Built

Full-stack conference quiz app for RSVP Baha Mar, built from scratch in one session. Live at https://rsvp-baha-mar-ai.netlify.app/

### App Flow
1. **index.html** — Capture form (name, email, phone, industry dropdown)
2. **quiz.html** — 20 yes/no questions, one at a time, CSS slide animation
3. **complete.html** — Confirmation page + free-form AI and group question submission
4. **dashboard.html** — Live presenter dashboard with bar charts, industry breakdown, focus overlay, AI insights

### Backend (Netlify Functions)
- **submit.mjs** — Validates POST, inserts to Supabase, sends results email (all 20 questions with checkmark/X, CTA buttons)
- **results.mjs** — Aggregates all responses for dashboard (polled every 10s)
- **insights.mjs** — Calls Claude Sonnet for 3-5 narrative observations (polled every 60s)
- **questions.mjs** — Saves free-form AI/group questions to existing response row, sends confirmation email

### Shared Modules
- **lib/questions.mjs** — Single source of truth for 20 questions, recommendations, trigger logic
- **lib/aggregate.mjs** — Per-question and per-industry yes/no aggregation
- **lib/supabase.mjs** — Shared client init

### Other
- **generate-qr.js** — `node generate-qr.js <URL>` produces QR code PNG for slide deck
- **QR code generated** for https://rsvp-baha-mar-ai.netlify.app/

## Decisions Made

- **Vanilla HTML/CSS/JS** — No frameworks, fast on conference Wi-Fi
- **Single Supabase table** (`responses`) — All data in one row per attendee including free-form questions
- **t2 is inverted** — "Are you spending hours on manual tasks?" triggers recommendation on "Yes" not "No"
- **Two emails, not one** — Results email sends on quiz submit (all 20 questions). Second email sends if user submits AI/group questions on complete page.
- **Results email shows all 20 questions** — Checkmark for Yes answers, X with recommendation for No answers. CTA mailto buttons for Alain and Travis at the bottom for anyone with No answers.
- **Questions email** — Lists user's AI and group questions with note that Alain/Travis will follow up via email
- **Dashboard focus overlay** — Fixed-position overlay with opacity transition, no DOM manipulation
- **AI/group questions on complete.html** — Not in the quiz flow; users can add multiple questions and submit after quiz
- **Questions append** — Submitting more questions adds to existing arrays, doesn't overwrite
- **Logo order** — AMS first, TMG second on all pages
- **AMS logo styling** — White background to match their brand

## Lessons Learned

- **Resend idempotency keys block retesting** — The original idempotency key (`rsvp-results/{email}`) cached the failed sandbox send for 24 hours, silently blocking all subsequent sends to the same email. Removed idempotency keys entirely since each quiz submission is unique. If idempotency is needed later, include a timestamp or response ID in the key.
- **Resend sandbox only delivers to account owner** — `onboarding@resend.dev` can only send to the email registered with Resend. Must verify your own domain first for real delivery.
- **Fire-and-forget hides errors** — The original async email send swallowed all errors silently. Switched to awaited send with error surfaced in the API response for debugging. Keep this pattern for now; revert to fire-and-forget after the event if latency matters.
- **Logo sizing is iterative** — Went through three rounds (48px → 72px → 144px). Start bigger next time for conference/projector contexts.

## Supabase Schema

```sql
CREATE TABLE responses (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at timestamptz DEFAULT now(),
  name text NOT NULL,
  email text NOT NULL,
  phone text,
  industry text NOT NULL,
  answers jsonb NOT NULL
);
ALTER TABLE responses ADD COLUMN ai_questions text[] DEFAULT '{}';
ALTER TABLE responses ADD COLUMN group_questions text[] DEFAULT '{}';
CREATE INDEX idx_responses_industry ON responses(industry);
CREATE INDEX idx_responses_created_at ON responses(created_at);
```
